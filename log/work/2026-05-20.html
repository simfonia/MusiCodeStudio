技術深挖 (Technical Deep Dive)

1. 參數派發的回補 (Fallback) 與離散映射

在 4OSC 中，部分參數 (如 filterType) 不是 AutomatableParameter，無法自動轉換 0-1 範圍。我在 ParameterDispatcher 中實作了手動映射：

if (paramID == "filterType") mappedValue = std::round(normalizedValue * 4.0f); // 0~4

這確保了從積木或前端傳來的 0-1 標準化值能正確對接到插件的內部狀態。

2. te::EditItemID 的穩定性

Tracktion Engine 的 EditItemID 在物件建立時生成，並儲存在 ValueTree 中。即使音軌在 Edit 中的順序改變，其 ID 保持不變。這對於前端 UI (如 React) 維護元件狀態至關重要，避免了「索引移位」導致的操作錯誤。

3. PluginWindowState::showWindowExplicitly()

在重構中發現 showWindow() 是 private 成員，正確的 UI 操作應呼叫 showWindowExplicitly() 以強制開啟插件原生視窗。
