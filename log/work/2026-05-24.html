2. 關鍵問題診斷與修復 Bug Fix

  • WebView2 黑畫面與渲染崩潰：

    現象： 出現 80004002 (E_NOINTERFACE) 錯誤，畫面呈現黑色或彈出下載錯誤。

    診斷： 系統更新與重啟修復了底層 GPU/WARP 衝突。代碼端加入「3 秒啟動禁運期」與「GPU 停用參數」作為防禦性設計。

    最終方案： 恢復至標準原生模式 (Hardware Accelerated)，保留注入守衛以確保載入穩定性。

  • 錄音後靜音與播放卡頓：

    修復： 在停止錄音後強制呼叫 ensureContextAllocated() 重建音訊圖表，並呼叫 setPosition(0) 歸位，消除 8 秒延遲。

3. 技術深挖 (Technical Deep Dive) Refactor

  • 模組化 Web 控制器：

    建立了 WebController.h，將 WebView2 的繁瑣配置、JS 事件發送、注入守衛邏輯從 MainWindow.cpp 中抽離。這大幅提升了主視窗代碼的可維護性。

  • Konva 顏色渲染限制：

    發現 Konva Rect 不支援 CSS 變數，必須硬編碼 Hex 顏色，解決了錄音色塊顯示為黑色的問題。
