MidiController 模組對照表

本模組負責全域 MIDI 設備管理、訊號監控與音軌路由，遵循 TE_Cookbook 規範。

功能 實作函數 通訊指令 / 事件 備註
設備掃描 refreshDevices() N/A 同步 JUCE 與 TE 設備清單，強制開啟 Monitor Mode。
清單獲取 getMidiInputsAsJson() get_midi_inputs 回傳目前所有啟用的實體 MIDI 設備。
音軌路由 setTrackInput() set_track_input 執行「五步驟路由」：Context -> Instance -> Target -> RecEnabled。
訊號監控 handleIncomingMidiMessage() midi_signal (Event) 30Hz 高頻推送，直接向 JUCE 註冊以確保低延遲與力度精確。

技術深挖：雙重更新同步模式 (Dual Update Pattern)

針對 Tracktion 內建插件（如 4OSC），為了解決 0-1 標準化數值與內部 Hz 數值的衝突，採用以下開發模式：

  • 第一重： 呼叫 param->setParameter() 以更新處理鏈與 UI。
  • 第二重： 呼叫 plugin->state.setProperty() 進行持久化，確保狀態正確寫入 ValueTree。
  • 回退： 若參數不屬於 AutomatableParameter，直接寫入 ValueTree。
