Tracktion Engine (TE) 開發手冊

本文檔粹取自 engine/tracktion_engine/examples，旨在提供 MusiCode Studio 開發所需的正確 API 呼叫模式與架構知識。

1. 核心觀念 (Core Concepts)

  • Engine: 全域單例，管理所有設備與資源。
  • Edit: 代表一個專案（Song/Session），包含所有軌道、Clip 與狀態。
  • Transport: 控制播放、停止、錄音與時間指標。
  • Track: 軌道，可分為 AudioTrack (含 MIDI) 與其它類型。

2. MIDI 處理與即時監聽 (Midi & Live Monitoring)

這是最容易出錯的部分。若順序錯誤，即便程式碼看起來正確也會靜音。

關鍵順序 (Order of Operations)：必須先分配播放上下文 (Context)，輸入設備的「實例 (Instance)」才會被建立。若在分配上下文前呼叫 getAllInputDevices()，將得到空列表。
步驟 關鍵 API / 邏輯 說明
1. 啟用設備 dm.deviceManager.setMidiInputDeviceEnabled(id, true) 在 JUCE 層級啟用實體設備。
2. 同步 TE dm.rescanMidiDeviceList() 讓 Tracktion Engine 重新掃描並封裝設備。
3. 建立上下文 edit->getTransport().ensureContextAllocated() 最重要！ 此時 TE 才會為 Edit 建立輸入實例。
4. 執行路由 instance->setTarget(...) 獲取 getAllInputDevices() 實例並指向軌道。
5. 開啟監聽 instance->setRecordingEnabled(trackID, true) 必須開啟此開關，訊號才會流過軌道。

驗證成功的程式碼模板 (Verified Pattern)

// 1. 確保設備在 JUCE 層已開啟
auto& dm = engine.getDeviceManager();
for (auto& dev : juce::MidiInput::getAvailableDevices())
    dm.deviceManager.setMidiInputDeviceEnabled (dev.identifier, true);
dm.rescanMidiDeviceList();

// 2. 準備上下文
edit->getTransport().ensureContextAllocated();

// 3. 掃描實例並路由
for (auto instance : edit->getAllInputDevices())
{
    if (instance->getInputDevice().getDeviceType() == te::InputDevice::physicalMidiDevice)
    {
        instance->setTarget (track->itemID, true, &edit->getUndoManager(), 0);
        instance->setRecordingEnabled (track->itemID, true); // 監聽必備
        
        // 診斷監聽狀態
        if (instance->isLivePlayEnabled (*track))
            DBG("Monitoring ACTIVE");
    }
}

3. 播放與傳輸 (Playback & Transport)

控制播放狀態的正確方式：

auto& transport = edit->getTransport();
if (transport.isPlaying())
    transport.stop(false, false);
else
    transport.play(false);
play(false) 中的 false 代表不使用回溯 (no looping / no rewind to start)。

4. 插件管理 (Plugins)

TE 插件不應依賴索引，應使用類型或名稱尋找。

任務 推薦做法
尋找特定插件 track->pluginList.findFirstPluginOfType<T>()
建立新插件 edit->getPluginCache().createNewPlugin(type, state)
同步參數 (UI -> DSP) parameter->setParameter(val, sendNotification)
任何涉及 ValueTree 或 Edit 狀態的變更，若不在 UI 執行緒，必須使用 juce::MessageManager::callAsync 包裹。

5. 診斷與調試 (Diagnostics)

  • 使用 edit->getTransport().ensureContextAllocated() 確保音訊上下文已建立。
  • 檢查 instance->isLivePlayEnabled(*track) 確認監聽是否真正生效。
